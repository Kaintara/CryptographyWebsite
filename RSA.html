RSA Encryption Algorithm

The Rivest–Shamir–Adleman (RSA) encryption method is used to generate a secure public key once a private key has been chosen, relying on Euler’s theorem and Bézout’s identity. These are two mathematical results that, when combined with large prime numbers, make it possible to construct key pairs with a unique one-way relationship: encryption with the public key can only be reversed by the corresponding private key.


What is Euler’s theorem

Euler’s theorm states: m𝚽(n) ≡ 1 (mod n) - but what is 𝚽(n) you ask? That is Euler’s Totient Function aka the Phi Function as 𝚽 is the greek letter 'phi'. This function outputs the number for any integer n the amount of coprimes n has, where m is coprime with n if their greatest common divisor (gcd) is equal to 1, gcd(m,n) = 1. For example 𝚽(10) = 4 as 1, 3, 7, and 9 all share the greatest common divisor of 1 with 10.

If you hadn't noticed already, the output of the phi function when prime numbers are inputted seem to be that number minus one. 𝚽(11) = 10 or 𝚽(11) = 11 - 1. This is due to the fact that prime numbers only share a common factor with 1 and themselves. Meaning all numbers less than p, a random prime, except 1 is coprime with p. Therefore 𝚽(p) = (p -1). This is extremely useful as to calculate the output of the phi function otherwise has a time complexity of O(n), making it extremely inefficient to find when dealing with large numbers that are not prime. What is Bézout’s identity?

Bezout’s identity states that where the integers a and b have the greatest common divisor of d ( gcd(a,b) = d ), they are integers which satisfy this equation - ay + bx = d. This can be used in addtion with Euler's Theorem to create a general formula for composite functions that are incredibly difficult to reverse engineer.

1 to the power of any positive integer is 1 therefore it can be rewritten to this: mk𝚽(n) ≡ 1 (mod n)

Multiply both sides by m to get this result - m*mk𝚽(n) ≡ 1*m (mod n)

mk𝚽(n) + 1 ≡ m (mod n)
