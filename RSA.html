RSA Encryption Algorithm

The Rivest–Shamir–Adleman (RSA) encryption method is used to generate a secure public key once a private key has been chosen, relying on Euler’s theorem and Bézout’s identity. These are two mathematical results that, when combined with large prime numbers, make it possible to construct key pairs with a unique one-way relationship: encryption with the public key can only be reversed by the corresponding private key.


What is Euler’s theorem

Euler’s theorm states: m𝚽(n) ≡ 1 (mod n) - but what is 𝚽(n) you ask? That is Euler’s Totient Function aka the Phi Function as 𝚽 is the greek letter 'phi'. This function outputs the number for any integer n the amount of coprimes n has, where m is coprime with n if their greatest common divisor (gcd) is equal to 1, gcd(m,n) = 1. For example 𝚽(10) = 4 as 1, 3, 7, and 9 all share the greatest common divisor of 1 with 10.

If you hadn't noticed already, the output of the phi function when prime numbers are inputted seem to be that number minus one. 𝚽(11) = 10 or 𝚽(11) = 11 - 1. This is extremely useful as to calculate the output of the phi function otherwise has a time complexity of O(n), making it extremely inefficient to find when dealing with large numbers that are not prime. What is Bézout’s identity?

Bezout’s identity states that where the integers a and b have the greatest common divisor of d ( gcd(a,b) = d ), they are integers which satisfy this equation - ay + bx = d. This can be used in addtion with Euler's Theorem to create a general formula for composite functions that are incredibly difficult to reverse engineer.


RSA Mathematical Proof!

Starting off by stating the two integers m and n as coprimes of each other, they can be inputted into Euler’s theorem giving this result - m𝚽(n) ≡ 1 (mod n).

Then using k as any positive integer, increase the power by k on both sides: mk𝚽(n) ≡ 1k (mod n)

1 to the power of any positive integer is 1 therefore it can be rewritten to this: mk𝚽(n) ≡ 1 (mod n)

Multiply both sides by m to get this result - m*mk𝚽(n) ≡ 1*m (mod n)

mk𝚽(n) + 1 ≡ m (mod n)

Bezout’s identity then can be used by stating k𝚽(n) + 1 = ed where gcd(𝚽(n),e) = 1, k and d are the integers which satisfy the equation.

Substituting e and d into the equation results in med ≡ m (mod n)

This leaves us with composite functions: f(m) = me mod n and g(m) = md mod n so that gf(m) = m.

This proof can then be applied as a method of asymmetric encryption where m is the message/data being encrypted, f(m) is the function to encrypt and g(m) is the function to decrypt. As f(m) = me mod n, the public key would be (e,n). Therefore, the private key would be (d,n) as g(m) = md mod n.

Generating Private and Public Keys

The proof above shows how the RSA works as an asymmetric method but it doesn’t show how exactly it is secure or how to generate e,d and n as they can’t just be any integer. For RSA to be a secure encryption method, a large product of two primes must be used as n. It is extremely difficult to calculate due to it being the product of two large primes ensuring security. As 𝚽(p) = (p-1), due to the phi function being multiplicative, 𝚽(pq) = (p-1)(q-1) where p and q are two random large prime numbers. So if n = pq, 𝚽(n) is easy to calculate as it simply equals (p-1)(q-1). As 𝚽(n) is necessary to derive d, p and q are not shared publicly making the private key (d,n) is near impossible to calculate as without p and q, 𝚽(n) is extremely difficult to compute.


To generate e and d, Bezout’s identity must be used as it defined the relationship between, e, d and 𝚽(n) in the proof to derive the encryption and decryption functions. As stated before, gcd(𝚽(n),e) = 1, therefore e needs to be a coprime to 𝚽(n). e can be any number coprime with 𝚽(n), between 1 and 𝚽(n) (exclusive) and be chosen at random. From the values of e and 𝚽(n), a range for d can be derived. As we know e is coprime with n, k𝚽(n) + 1 = de which implies 1 ≡ de (mod 𝚽(n)). As you already know, 𝚽(n) and e so you can choose any value for d which satisfies this equation.

This leaves the public key as (19,32). To derive d, a value that satisfies this equation, 1 ≡ 19d (mod 32) needs to be calculated.
