Diffie-Hellman Key Exchange Algorithm

Diffie-Hellman key exchange is a method for two people to establish a shared private key while communicating over a public channel. Its security relies on modular exponentiation — a type of calculation that is easy to perform in one direction but extremely difficult to reverse without special knowledge. Since modular exponentiation is built on modular arithmetic, we’ll need to explore that first.

What is Modular Arithmetic?

Modular arithmetic is a system of arithmetic for integers, where numbers "wrap around" after reaching a certain value called the modulus. A good example is a 12-hour clock: it only shows numbers from 1 to 12, so after 12, it wraps around to 1 again. In modular terms, this means 13 ≡ 1 (mod 12), 14 ≡ 2 (mod 12) and so on. This states both sides leave the same remainder when divided by 12 as the modulo operation returns the remainder after division. Because of this the result of the modulo operation is always between 0 and one less than the modulus which causes the numbers to ‘wrap around’. Using the clock below and the modulus operation, you can convert the time from 24-hour to 12-hour format demonstrating this example.

Equivalent to 13 mod 12, therefore it is 1 pm.

This shows that modular exponentiation is just the application of modular arithmetic and exponents. Where a base number is raised to a power of an exponent and the modulo operation is then applied to it. This can be written as be mod n where b is the base, e is the exponent, and n is the modulus.

How Diffie-Hellman Works

The two users wanting to create a private key would both choose a number as their secret key, x and y which they keep to themselves. Then they agree on a base, b and a modulus, m, which they share publicly. Both users apply their secret key as an exponent to the shared base and then share the remainder of this value divided by the shared modulus publicly. So bx (mod m) and by (mod m) are exchanged between the users. They then use this value as their new base, using their own secret key and exponent again to calculate their own shared private key. So user 1 would do (by)x (mod m) and user 2 would do (bx)y (mod m) which are equal. bxy (mod m) becomes the shared private key.


User A

~ Chooses the secret key 8

~ Calculates 48 mod 17

~ Shares publicly the value 1 (as 1 ≡ 48 (mod 17))

~ Uses shared value as new base, calculating 138 mod 17

Both Users agree on a modulus of 17 and base of 4.

User B

~ Chooses the secret key 15

~ Calculates 415 mod 17

~ Shares publicly the value 13 (as 13 ≡ 48 (mod 17))

~ Uses shared value as new base, calculating 115 mod 17

User A and User B both return the same value 1 which becomes their shared private key.

How Diffie-Hellman is Secure

Diffie-Hellman is secure due to the Discrete Logarithm Problem. This problem suggests there is no efficient way to calculate the modular exponents x or y using the base, modulus or using the resulting value, bx (mod m). As these are the values shared publicly that could be intercepted, the discrete logarithm problem stops the resulting value from being inversed to calculate the exponents which are necessary to calculate the resulting shared private key.

Picking a Good Base and Modulus!

Additionally, the example used a base of 1.No matter what positive exponent that is used, it will always return 1 even after the modulo operation has been applied to it. A good choice of a base would be the primitive root modulo n. This is a number, when used as a base, which returns all possible results that can be outputted by the modulo operation using n as the modulus. There are only primitive roots for n = 1,2,4, pk or 2pk where p is an odd prime. This is another reason why choosing a large prime for the modulus is useful.
